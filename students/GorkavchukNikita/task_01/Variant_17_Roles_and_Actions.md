# Вариант 17 — Доставка «Где моя посылка?» — Роли и действия

Краткое описание

MVP: Order, TrackingEvent (Event), Address/Location, User. UI: публичный трекинг по `orderCode`, для пользователя — «мои заказы», для админа — управление справочниками и заказами. Приёмка (MVP): публичный трекинг.

Роли и основные действия

- Гость (public)
  - Ввод `orderCode` и просмотр статуса/таймлайна (GET /api/tracking/:code)
  - Переход на регистрацию/вход

- Пользователь (user)
  - Регистрация/вход, просмотр профиля
  - Просмотр списка «мои заказы» (GET /api/orders/my)
  - Публичный трекинг (как гость)

- Администратор (admin)
  - Вход в админку (POST /api/admin/signin)
  - CRUD пользователей (включая reset-password)
  - CRUD справочников: страны/города/локации
  - CRUD заказов
  - Добавление/редактирование/удаление событий трекинга (Event)

Примеры сценариев

- Админ создаёт локации (warehouse → sort_center → pickup_point) → создаёт заказ → система выдаёт `orderCode`.
- Пользователь отправляет `orderCode` другу/клиенту → тот открывает главную и видит «где посылка».
- Админ добавляет событие `in_transit` → order.status и currentLocation обновляются, в публичном трекинге появляется новое событие.
