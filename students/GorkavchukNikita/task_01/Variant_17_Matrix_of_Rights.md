# Вариант 17 — Матрица прав (операция × роль)

Операции: Публичный трекинг, Работа с профилем, Просмотр «Мои заказы», Управление справочниками (страны/города/локации), Управление заказами, Управление событиями, Управление пользователями.

Роли: Гость (public), Пользователь (user), Администратор (admin)

Таблица (кто может что)

- Публичный трекинг (GET /api/tracking/:code): Гость, Пользователь, Администратор
- Регистрация/вход (POST /api/user/signup|signin): Гость
- Профиль (GET /api/user/profile): Пользователь, Администратор
- «Мои заказы» (GET /api/orders/my): Пользователь, Администратор (если залогинен под admin)

- CRUD стран ( /api/admin/country ): Администратор
- CRUD городов ( /api/admin/city ): Администратор
- CRUD локаций/адресов ( /api/admin/location ): Администратор

- CRUD заказов ( /api/admin/order ): Администратор
- Управление событиями ( /api/admin/order/:oid/event ): Администратор

- CRUD пользователей ( /api/admin/users ): Администратор
- Сброс пароля пользователя ( /api/admin/users/:id/reset-password ): Администратор

Примечания:

- Авторизация реализована через cookie `jwt`. Для /api/admin/* дополнительно проверяется роль admin.
- Публичный трекинг — обязательная приёмка MVP: запрос по коду должен работать без авторизации.
